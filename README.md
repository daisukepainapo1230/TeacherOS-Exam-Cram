# TeacherOS Exam Cram

東京都の小学校教員採用試験に向けて、「小学校全科」と「論文」を短時間で学習する静的Webアプリです。iPhone・iPadのSafariを優先して設計しています。小学校全科はタップで解ける4択問題、論文練習はテーマ・キーワード・実践例を選ぶテンプレート式です。

小学校全科は、東京都公立学校教員採用候補者選考の公式過去問PDFの出題形式・出題分野・難易度を参考にしたオリジナル類題です。過去問本文の丸写しではありません。初期データとして、国語、社会、算数、理科、生活、音楽、図画工作、家庭、体育、外国語、特別の教科 道徳、総合的な学習の時間、特別活動の13教科・各5問、合計65問を収録しています。

学習記録画面から、問題データを次のCSV形式でコピーまたはダウンロードできます。

```csv
教科,問題,選択肢A,選択肢B,選択肢C,選択肢D,正解,解説
```

## 公開に使うファイル

専用のGitHubリポジトリを作り、次の4ファイルをリポジトリ直下へ置いてください。

- `index.html`
- `manifest.json`
- `service-worker.js`
- `README.md`

外部ライブラリ、ログイン、独自サーバーは不要です。学習記録、苦手問題、論文構成メモは、利用端末のSafariの `localStorage` に保存されます。

## 1. GitHub Pagesで公開する手順

1. GitHubへサインインし、TeacherOS Exam Cram専用の新しいリポジトリを作成します。GitHub Freeを使う場合は公開リポジトリにします。
2. 上記4ファイルをリポジトリの一番上の階層へアップロードし、`main` ブランチへコミットします。
3. リポジトリの `Settings` を開きます。
4. 左側の `Pages` を開きます。
5. `Build and deployment` の `Source` で `Deploy from a branch` を選びます。
6. Branchを `main`、フォルダを `/(root)` にして `Save` を押します。
7. 公開処理が終わるまで数分待ちます。`Pages` 画面に表示されたURLを開きます。

プロジェクト用サイトのURLは通常、次の形になります。

```text
https://ユーザー名.github.io/リポジトリ名/
```

公開元はブランチのルートまたは `/docs` を選べますが、このアプリは `main` の `/(root)` を前提にしています。詳しくは[GitHub公式の公開元設定手順](https://docs.github.com/en/pages/getting-started-with-github-pages/configuring-a-publishing-source-for-your-github-pages-site)を確認してください。

## 2. iPhone・iPadのSafariで開く手順

1. GitHub Pagesの公開URLをコピーします。
2. iPhoneまたはiPadでSafariを開きます。LINE、Gmail、Xなどのアプリ内ブラウザではなくSafariを使ってください。
3. アドレス欄へURLを貼り付けて開きます。
4. ホーム画面と学習メニューが表示されることを確認します。
5. 最初の一度は通信できる状態でページを最後まで読み込みます。これにより、オフライン利用に必要な基本ファイルがキャッシュされます。

学習記録は端末とURLごとに保存されます。別のiPhone・iPadへ自動同期はされません。Safariの履歴とWebサイトデータを消去すると、学習記録も消えることがあります。

## 3. ホーム画面に追加する手順

1. 公開URLをSafariで開きます。
2. Safariの共有ボタン（四角から上向き矢印が出ているアイコン）を押します。
3. メニューをスクロールし、`ホーム画面に追加` を選びます。
4. 表示名を確認して、右上の `追加` を押します。
5. 以後はホーム画面の `Exam Cram` アイコンから起動します。

ホーム画面から起動すると、Safariの通常タブよりもアプリに近い外観で表示されます。追加後に更新が反映されない場合は、Safariで公開URLを一度開き直してからホーム画面版を再起動してください。

## 4. 開けない場合の確認ポイント

- URLが `https://` で始まっているか確認してください。
- `index.html`、`manifest.json`、`service-worker.js` がリポジトリ直下にあり、ファイル名の大文字・小文字が一致しているか確認してください。
- GitHubの `Settings` → `Pages` で、公開元が `main` / `/(root)` になっているか確認してください。
- GitHubの `Actions` でPagesの公開処理が完了しているか確認してください。
- 404になる場合は、URLにリポジトリ名が含まれているか確認してください。
- アプリ内ブラウザではなくSafariで開いてください。
- Safariのプライベートブラウズでは保存が維持されない場合があります。通常ブラウズを使ってください。
- 古い画面が表示される場合はSafariでページを再読み込みし、それでも直らなければ `設定` → `Safari` → `詳細` → `Webサイトデータ` から対象サイトのデータを削除して開き直してください。この操作では対象サイトの学習記録も削除されます。
- ヘッダーと下部ナビだけ表示され、中央が空白になる場合は、古いキャッシュまたはJavaScriptの途中停止が疑われます。最新版では中央に安全表示と `読み込みOK` 表示が出るようにしています。Safariで公開URLを再読み込みし、必要に応じて対象サイトのWebサイトデータを削除してください。
- コンテンツブロッカーやVPNを一時的に無効にすると改善する場合があります。
- オフラインで開く前に、オンライン状態で少なくとも一度は正常に起動してください。iOSの空き容量が少ない場合など、OSの判断でキャッシュが削除されることがあります。

## 5. ファイルアプリから直接開く方法について

ファイルアプリから直接開く方法は非推奨です。

iCloud DriveやiOS・iPadOSのファイルアプリから `index.html` を直接開く方法は非推奨です。

ファイルアプリのプレビューでは、JavaScript、`localStorage`、manifest、Service WorkerがSafariの通常のWebページと同じように動作しないことがあります。必ずGitHub Pagesへ公開し、Safariから `https://` のURLとして開いてください。

## オフライン利用について

`service-worker.js` は、初回のオンライン起動時にアプリ本体とmanifestを端末へキャッシュします。その後は通信できない状況でも基本機能を起動できます。ただし、iOS・iPadOSがストレージを整理した場合はキャッシュが削除されることがあるため、完全な永続保存を保証するものではありません。
